# Paraphrase robustness: how much do dataset-phrased questions inflate the scores?

## Why this evaluation exists

The standard retrieval evaluation queries the index with the dataset's **own** questions.
But those questions and their answers were generated as pairs, so they share vocabulary:
measured over 300 sampled questions, **59% of a question's words (4+ characters) also appear
in its own gold answer**, versus **4%** for a random answer.

That makes the retrieval task artificially easy. Real users do not phrase questions the way
the dataset does. To quantify the gap, each question was rewritten into colloquial
user language by a local LLM (`llama3.2:3b`), and the *same* sample was evaluated twice.

Example rewrites:

| Dataset phrasing | Paraphrased as a user would type it |
|---|---|
| What are the risks of high blood pressure? | my blood pressure is high, what bad stuff can happen? |
| What are the symptoms of a panic attack? | what happens when im freaking out really bad and my heart is racing? |
| What is the recovery time for a tummy tuck? | how long does it take to recover from getting my stomach sucked in? |
| Can someone with Marfan syndrome still drive? | can people with a rare genetic thing that affects their body still be behind the wheel? |

## Results (n = 99, same questions in both columns)

| Method | Recall@5 (dataset phrasing) | Recall@5 (paraphrased) | Relative drop |
|---|---|---|---|
| BM25 (sparse keyword) | 0.626 | **0.040** | **−93.5%** |
| Dense + FAISS HNSW | 0.758 | **0.253** | **−66.7%** |

| Method | MRR (dataset phrasing) | MRR (paraphrased) |
|---|---|---|
| BM25 | 0.467 | 0.021 |
| Dense | 0.601 | 0.118 |

*(Recall on dataset phrasing differs slightly from the main 500-question run because this
uses a 99-question subsample.)*

## Findings

**1. Both methods were heavily inflated by shared vocabulary.** Neither number survives
realistic phrasing. Absolute recall from the standard evaluation should not be read as
production performance.

**2. BM25 collapses almost completely (−93.5%).** This is expected and mechanistically
clear: BM25 matches literal terms. Once the paraphrase removes the shared words, there is
nothing left to match on. Its apparent competence in the standard evaluation was almost
entirely an artefact of vocabulary overlap.

**3. Dense retrieval's advantage is far larger than the standard evaluation suggested.**

| Query style | Dense ÷ BM25 (Recall@5) |
|---|---|
| Dataset phrasing | 1.2× |
| Paraphrased (realistic) | **6.3×** |

This reverses the framing of the original result. The headline comparison (0.730 vs 0.566)
*understated* the value of semantic retrieval, because dataset phrasing hands BM25 the exact
keywords it needs. On queries that look like real user input, embeddings are not a marginal
1.2× improvement — they are the difference between a system that works occasionally and one
that essentially does not work at all.

**4. Dense retrieval at 0.253 Recall@5 is still poor in absolute terms.** On realistically
phrased questions, the correct passage reaches the generator only about a quarter of the
time. This is the honest ceiling on the current system's real-world quality, and it points
directly at the improvements not yet implemented: hybrid search, cross-encoder re-ranking,
and a stronger embedding model.

## Caveats

- **This is a pessimistic bound.** The paraphraser sometimes strips the key clinical term
  entirely ("Marfan syndrome" → "a rare genetic thing"; "hypothalamus" → "this tiny part of
  my brain"), leaving almost no signal to retrieve on. Real users frequently *do* know their
  diagnosis or the medical term. True performance likely sits between the two columns:
  **roughly 0.25–0.76 Recall@5**, and closer to the middle for users who know some terminology.
- **Paraphrase quality was not independently validated.** Rewrites were spot-checked by hand
  and preserved meaning, but were generated by the same 3B model used elsewhere in the
  pipeline; a rewrite that lost meaning would unfairly penalise retrieval.
- **Single sample, single seed** (n = 99, seed 42). No confidence intervals computed.

## Reproducing

```bash
python evaluate_paraphrase.py -n 100          # uses cached paraphrases if present
python evaluate_paraphrase.py -n 100 --regenerate   # force new paraphrases
```

Paraphrases are cached in `results/retrieval/paraphrases.json`; raw metrics in
`results/retrieval/paraphrase.json`.
